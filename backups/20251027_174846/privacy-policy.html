1. Introduction

EZer ("we", "our", "us", or "the App") is a personal finance management application developed for Android and iOS platforms. EZer helps you track expenses, manage budgets, scan receipts, import transactions, and gain insights into your financial health.

This Privacy Policy explains:

  • What personal data we collect and why
  • How we store and protect your data
  • Your rights regarding your personal information
  • How to exercise your privacy rights

By downloading, installing, or using EZer, you agree to this Privacy Policy. If you do not agree, please do not use the App.

2. Information We Collect

EZer collects different types of information depending on the features you choose to use. All data collection requires your explicit consent.

2.1 Financial Data (Required for Core Functionality)

When you use EZer, we collect and store the following financial information locally on your device:

  • Transactions: Date, amount, merchant name, category, payment method, notes
  • Accounts: Account names, types (bank, wallet, credit card), balances
  • Budgets: Budget categories, limits, time periods, spending alerts
  • Bills: Biller names, amounts, due dates, payment history, recurrence settings
  • Goals: Savings goal names, target amounts, deadlines, progress tracking
  • Categories: Custom categories and subcategories you create
  • Receipts: Images of receipts you scan or attach to transactions

2.2 SMS Messages (Optional - Android Only)

Android users only: If you grant SMS permission, EZer will:

  • Read SMS messages from known financial institutions (banks, payment apps, credit card companies)
  • Extract transaction details (amount, merchant, date, account) to automatically create transaction records
  • Process SMS messages locally on your device - messages are never transmitted to external servers
  • Only read messages that match banking transaction patterns

Important: SMS permission is entirely optional. You can use all core features of EZer without granting SMS access. This permission can be revoked at any time through the Privacy Dashboard or device settings.

2.3 Camera and Storage Access (Optional)

If you enable receipt scanning features, we request:

  • Camera Access: To capture photos of receipts
  • Photo Library/Storage Access: To select existing receipt images from your device

Receipt images are stored encrypted in your device's local database and processed using on-device OCR (Optical Character Recognition) powered by Google ML Kit.

2.4 Notification Access (Optional - Android Only)

If you enable automatic transaction detection on Android, EZer can read notifications from banking apps to automatically log transactions. All notification processing happens locally on your device.

2.5 Device Information (Automatically Collected)

We collect minimal technical information for app functionality and troubleshooting:

  • Device model and manufacturer
  • Operating system version (iOS/Android version)
  • App version number
  • Device language and region settings
  • Screen resolution and display metrics (for UI optimization)

This information does not include device identifiers like IMEI or serial numbers.

2.6 Usage Analytics (Optional - Opt-In Only)

If you explicitly enable analytics in Settings, we collect anonymous usage data:

  • Screen views and navigation patterns (e.g., "Budget screen viewed")
  • Feature usage frequency (e.g., "Receipt scan used 5 times")
  • App performance metrics (loading times, errors)
  • Crash reports with stack traces

What we DO NOT collect in analytics:

  • Transaction amounts or balances
  • Merchant names or transaction descriptions
  • Account names or numbers
  • Any personally identifiable financial data

2.7 Biometric Data (Processed Locally)

If you enable biometric authentication (fingerprint, Face ID), biometric data is:

  • Processed entirely by your device's operating system (iOS Keychain, Android Keystore)
  • Never accessed, stored, or transmitted by EZer
  • Used only to unlock the app - we never receive the actual biometric data

3. How We Store Your Data

üõ°Ô∏è Local-First, Encrypted Storage

Your financial data never leaves your device unless you explicitly choose to export it or enable cloud backup. All data is stored in an encrypted SQLite database using SQLCipher encryption.

3.1 Database Encryption

EZer uses industry-standard encryption:

  • Database Encryption: SQLCipher with AES-256-GCM encryption
  • Encryption Keys: Stored in device-specific secure storage (Android Keystore, iOS Keychain)
  • Key Generation: Cryptographically secure random key generation
  • No Master Password: Encryption keys are hardware-backed and device-specific

3.2 Secure Storage Locations

  • Android: App-specific private storage with Android Keystore encryption
  • iOS: App sandbox with iOS Keychain encryption and biometric protection
  • Receipt Images: Encrypted and stored in the app's private cache directory

3.3 Root/Jailbreak Detection

EZer implements security checks using freeRASP to detect:

  • Rooted Android devices
  • Jailbroken iOS devices
  • App debugging and tampering
  • Emulator/simulator environments

If security risks are detected, EZer will warn you and may restrict sensitive features to protect your data.

6. SMS Permissions (Android Only)

6.1 Why We Request SMS Permission

SMS auto-import is a convenience feature that saves you time by automatically detecting banking transaction SMS and creating transaction records. This is especially useful in regions like India where SMS is the primary transaction notification method.

6.2 How SMS Processing Works

  1. You grant SMS permission through Android's permission dialog
  2. EZer scans SMS messages for banking transaction patterns
  3. Only messages from known financial institutions are processed (banks, UPI apps, wallets)
  4. Transaction details (amount, merchant, date) are extracted using on-device parsing
  5. A transaction draft is created for your review
  6. SMS messages are never transmitted to external servers

6.3 SMS Data Retention

  • EZer does not store full SMS message content
  • Only extracted transaction details (amount, merchant, date) are stored
  • Original SMS messages remain in your device's messaging app

6.4 Revoking SMS Permission

You can revoke SMS access at any time:

  • Via the Privacy Dashboard in EZer Settings
  • Via Android Settings ‚Üí Apps ‚Üí EZer ‚Üí Permissions ‚Üí SMS ‚Üí Deny

Revoking SMS permission does not affect any other app functionality.

7. Notification Permissions

7.1 In-App Notifications

EZer sends local notifications for:

  • Bill Reminders: Upcoming bill due dates
  • Budget Alerts: When you approach or exceed budget limits
  • Goal Milestones: Progress updates on savings goals
  • Spending Insights: Weekly or monthly spending summaries

All notifications are generated locally on your device. You can customize or disable notifications in Settings.

7.2 Notification Listener (Android Only - Optional)

If you enable automatic transaction detection from banking app notifications:

  • EZer can read notifications from banking apps (like Google Pay, PhonePe, Paytm)
  • Transaction notifications are parsed locally to auto-create transactions
  • Notification content is never transmitted to external servers
  • Only banking app notifications are processed - other app notifications are ignored

8. Camera and Storage Permissions

8.1 Camera Access

Camera permission is used exclusively for:

  • Capturing receipt photos
  • Scanning documents related to financial transactions

EZer does not access your camera for any other purpose, nor does it record videos or access photo metadata.

8.2 Photo Library Access

Storage/photo library permission allows you to:

  • Select existing receipt images from your photo library
  • Export financial reports to your device storage
  • Save backup files locally

8.3 Receipt Image Processing

When you scan a receipt:

  1. Image is captured or selected from your photo library
  2. OCR processing happens on-device using Google ML Kit (no cloud upload)
  3. Extracted text (merchant, amount, date) is saved to your encrypted database
  4. Receipt image is encrypted and stored locally
  5. Original photo in your photo library is not modified or deleted

9. Machine Learning and AI Features

9.1 On-Device Machine Learning

EZer uses machine learning for:

  • Automatic Categorization: Suggesting categories for transactions based on merchant names
  • Spending Insights: Identifying unusual spending patterns and trends
  • Budget Predictions: Forecasting future spending based on historical data
  • OCR Enhancement: Improving receipt text recognition accuracy

9.2 Privacy-Preserving ML

All machine learning in EZer is privacy-preserving:

  • ML models run entirely on your device (TensorFlow Lite)
  • Your transaction data is never sent to external servers for training
  • Models are pre-trained and included in the app
  • Personalized learning happens locally using on-device data

10. Cloud Backup (Optional)

10.1 How Cloud Backup Works

If you enable cloud backup:

  1. You authenticate with Google Drive (Android) or iCloud (iOS)
  2. EZer creates an encrypted backup of your database
  3. Backup is encrypted using your device encryption key before upload
  4. Encrypted backup is stored in your personal cloud storage account
  5. Only you can decrypt and restore the backup on your device

10.2 What's Included in Backups

  • All transaction records
  • Accounts, budgets, bills, goals
  • Categories and custom settings
  • Receipt images (encrypted)
  • App preferences

10.3 Backup Security

  • Backups are encrypted using AES-256 before upload
  • Encryption key remains on your device (never uploaded)
  • Google/Apple cannot decrypt your backups
  • Backups are accessible only through your authenticated account
