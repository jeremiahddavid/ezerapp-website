1. Introduction

EZer ("we", "our", "us", or "the App") is a personal finance management application developed for Android devices. EZer helps you track expenses, manage budgets, scan receipts, import transactions via SMS, and gain insights into your financial health.

This Privacy Policy explains:

  • What personal data we collect and why
  • How we store and protect your data on Android
  • Android-specific permissions (SMS, notifications, storage)
  • Your rights regarding your personal information
  • How to exercise your privacy rights

By downloading, installing, or using EZer on Android, you agree to this Privacy Policy. If you do not agree, please do not use the App.

2. Information We Collect

EZer collects different types of information depending on the features you choose to use. All data collection requires your explicit consent.

2.1 Financial Data (Required for Core Functionality)

When you use EZer, we collect and store the following financial information locally on your Android device:

  • Transactions: Date, amount, merchant name, category, payment method, notes
  • Accounts: Account names, types (bank, wallet, credit card), balances
  • Budgets: Budget categories, limits, time periods, spending alerts
  • Bills: Biller names, amounts, due dates, payment history, recurrence settings
  • Goals: Savings goal names, target amounts, deadlines, progress tracking
  • Categories: Custom categories and subcategories you create
  • Receipts: Images of receipts you scan or attach to transactions

2.2 SMS Messages (Optional - Android Only)

üì± Android-Specific Feature

SMS permission is entirely optional and Android-exclusive. This feature is useful in regions where banking SMS is a common transaction notification method.

If you grant SMS permission on Android, EZer will:

  • Read SMS messages from known financial institutions (banks, payment apps, digital wallets, credit card companies)
  • Extract transaction details (amount, merchant, date, account) to automatically create transaction records
  • Process SMS messages locally on your device - messages are never transmitted to external servers
  • Only read messages that match banking transaction patterns from recognized financial institutions

‚ö†Ô∏è Important

  • ‚úì SMS permission is entirely optional - you can use all core features without it
  • ‚úì Can be revoked anytime via Privacy Dashboard or Android Settings
  • ‚úì We do NOT read personal messages - only banking transaction SMS
  • ‚úì SMS content never leaves your device

2.3 Camera and Storage Access (Optional)

If you enable receipt scanning features, we request:

  • Camera Access: To capture photos of receipts
  • Photo Library/Storage Access: To select existing receipt images from your device (Android 13+ uses scoped storage)

Receipt images are stored encrypted in your device's local database and processed using on-device OCR (Optical Character Recognition) powered by Google ML Kit. No images are uploaded to external servers.

2.4 Notification Access (Optional - Android Only)

üîî Android Notification Listener

If you enable automatic transaction detection, EZer can read notifications from banking and payment apps to automatically log transactions. All processing is done locally on your device.

2.5 Device Information (Automatically Collected)

We collect minimal technical information for app functionality and troubleshooting:

  • Device model and manufacturer (e.g., Samsung Galaxy S23, Google Pixel 8)
  • Android OS version (e.g., Android 14)
  • App version number
  • Device language and region settings
  • Screen resolution and display metrics (for UI optimization)

This information does not include device identifiers like IMEI, serial numbers, or Android Advertising ID.

2.6 Usage Analytics (Optional - Opt-In Only)

If you explicitly enable analytics in Settings, we collect anonymous usage data:

  • Screen views and navigation patterns (e.g., "Budget screen viewed")
  • Feature usage frequency (e.g., "Receipt scan used 5 times")
  • App performance metrics (loading times, errors)
  • Crash reports with stack traces

üö´ What we DO NOT collect in analytics:

  • Transaction amounts or balances
  • Merchant names or transaction descriptions
  • Account names or numbers
  • Any personally identifiable financial data

2.7 Biometric Data (Processed Locally)

If you enable biometric authentication (fingerprint, face unlock), biometric data is:

  • Processed entirely by Android's BiometricPrompt API and Android Keystore
  • Never accessed, stored, or transmitted by EZer
  • Used only to unlock the app - we never receive the actual biometric data

3. How We Store Your Data (Android)

üõ°Ô∏è

Local-First, Encrypted Storage on Android

Your financial data never leaves your Android device unless you explicitly choose to export it or enable cloud backup. All data is stored in an encrypted SQLite database using SQLCipher encryption with Android Keystore.

3.1 Database Encryption (Android Keystore)

EZer uses industry-standard encryption on Android:

  • Database Encryption: SQLCipher with AES-256-GCM encryption
  • Encryption Keys: Stored in Android Keystore (hardware-backed on supported devices)
  • Key Generation: Cryptographically secure random key generation using Android SecureRandom
  • No Master Password: Encryption keys are device-specific and bound to Android Keystore
  • StrongBox Support: Uses hardware security module on compatible devices (Pixel, Samsung flagship)

3.2 Secure Storage Locations (Android)

  • Database: App-specific private storage (/data/data/com.ezer.personal_finance/databases/) - inaccessible to other apps
  • Encryption Keys: Android Keystore (system-level secure enclave)
  • Receipt Images: Encrypted and stored in app's private cache directory
  • Shared Preferences: Encrypted using EncryptedSharedPreferences with Android Keystore

3.3 Root Detection (Android)

üîç Security Checks

EZer implements security checks using freeRASP to detect:

  • Rooted Android devices (Magisk, SuperSU, KingRoot)
  • Custom ROMs with su binaries
  • App debugging and tampering
  • Emulator/virtual machine environments
  • Xposed Framework or Frida hooks

If security risks are detected, EZer will warn you and may restrict sensitive features to protect your data.

6. SMS Permissions (Android Only)

üì± Android-Exclusive Feature

SMS auto-import is a convenience feature available only on Android in supported regions where banking SMS notifications are commonly used for transaction alerts.

6.1 Why We Request SMS Permission

SMS auto-import saves you time by automatically detecting banking transaction SMS from:

  • Banks and financial institutions
  • Payment apps and digital wallets
  • Credit card companies
  • Other recognized financial service providers

6.2 How SMS Processing Works on Android

  1. You grant READ_SMS permission through Android's system permission dialog
  2. EZer scans SMS messages for banking transaction patterns using regex
  3. Only messages from known financial institutions are processed
  4. Transaction details (amount, merchant, date) are extracted using on-device parsing
  5. A transaction draft is created for your review
  6. SMS messages are never transmitted to external servers

6.3 SMS Data Retention

  • EZer does not store full SMS message content
  • Only extracted transaction details (amount, merchant, date) are stored in encrypted database
  • Original SMS messages remain in your Android messaging app (unchanged)

6.4 Revoking SMS Permission on Android

You can revoke SMS access at any time:

  • Via the Privacy Dashboard in EZer Settings
  • Via Android Settings ‚Üí Apps ‚Üí EZer ‚Üí Permissions ‚Üí SMS ‚Üí Deny

Revoking SMS permission does not affect any other app functionality.

7. Notification Permissions (Android)

7.1 In-App Notifications

EZer sends local notifications on Android for:

  • Bill Reminders: Upcoming bill due dates (using Android AlarmManager)
  • Budget Alerts: When you approach or exceed budget limits
  • Goal Milestones: Progress updates on savings goals
  • Spending Insights: Weekly or monthly spending summaries

All notifications are generated locally using Android's NotificationManager. You can customize notification channels in Android Settings ‚Üí Apps ‚Üí EZer ‚Üí Notifications.

7.2 Notification Listener (Optional - Android Only)

üîî Advanced Android Feature

If you enable automatic transaction detection from banking app notifications:

  • EZer can read notifications from banking and payment apps
  • Transaction notifications are parsed locally to auto-create transactions
  • Notification content is never transmitted to external servers
  • Only financial app notifications are processed - other app notifications are ignored
  • Requires BIND_NOTIFICATION_LISTENER_SERVICE permission (granted via Android Settings)

8. Camera and Storage Permissions (Android)

8.1 Camera Access (Android)

Camera permission (android.permission.CAMERA) is used exclusively for:

  • Capturing receipt photos using Android Camera2 API
  • Scanning documents related to financial transactions

EZer does not access your camera for any other purpose, nor does it record videos or access photo metadata (EXIF data).

8.2 Photo Library Access (Android Storage)

Storage permission allows you to:

  • Android 13+: Uses scoped storage (READ_MEDIA_IMAGES) - access only to selected images
  • Android 12 and below: READ_EXTERNAL_STORAGE for selecting receipt images
  • Export financial reports to Downloads folder
  • Save backup files locally

8.3 Receipt Image Processing (On-Device OCR)

When you scan a receipt on Android:

  1. Image is captured using Android Camera API or selected from photo library
  2. OCR processing happens on-device using Google ML Kit (no cloud upload)
  3. Extracted text (merchant, amount, date) is saved to your encrypted SQLite database
  4. Receipt image is encrypted and stored in app's private storage
  5. Original photo in your photo library is not modified or deleted

9. Machine Learning and AI Features (On-Device)

9.1 On-Device Machine Learning (TensorFlow Lite)

EZer uses machine learning on your Android device for:

  • Automatic Categorization: Suggesting categories for transactions based on merchant names
  • Spending Insights: Identifying unusual spending patterns and trends
  • Budget Predictions: Forecasting future spending based on historical data
  • OCR Enhancement: Improving receipt text recognition accuracy

9.2 Privacy-Preserving ML (TensorFlow Lite on Android)

ü§ñ On-Device AI

All machine learning in EZer is privacy-preserving and runs entirely on your Android device:

  • ML models run using TensorFlow Lite (optimized for Android)
  • Your transaction data is never sent to external servers for training
  • Pre-trained models are included in the APK
  • Personalized learning happens locally using on-device data
  • Uses Android NNAPI for hardware acceleration on supported devices

10. Cloud Backup (Optional - Google Drive)

10.1 How Google Drive Backup Works on Android

If you enable cloud backup:

  1. You authenticate with Google Drive using Google Sign-In
  2. EZer creates an encrypted backup of your SQLite database
  3. Backup is encrypted using your Android Keystore encryption key before upload
  4. Encrypted backup is stored in your personal Google Drive account
  5. Only you can decrypt and restore the backup on your Android device

10.2 What's Included in Android Backups

  • All transaction records from SQLite database
  • Accounts, budgets, bills, goals
  • Categories and custom settings
  • Receipt images (encrypted)
  • App preferences (SharedPreferences)

10.3 Backup Security (Android Keystore)

  • Backups are encrypted using AES-256 before upload
  • Encryption key remains in Android Keystore (never uploaded)
  • Google cannot decrypt your backups
  • Backups are accessible only through your authenticated Google account

12. Data Retention and Deletion (Android)

12.1 How Long We Retain Data on Android

  • Financial Data: Retained on your Android device until you delete it or uninstall the app
  • Receipt Images: Stored in app cache until you delete the associated transaction or clear cache
  • Consent Logs: Audit logs retained for 2 years for compliance purposes
  • Google Drive Backups: Retained in your Google Drive until you delete them

12.2 Automatic Data Cleanup on Android

EZer automatically cleans up:

  • Temporary OCR cache files (cleared after 7 days)
  • Deleted transactions (permanently removed after 30 days)
  • Old notification logs (kept for 90 days for audit purposes)

12.3 Permanent Data Deletion on Android

To permanently delete all your data from your Android device:

  1. Open EZer ‚Üí Settings ‚Üí Privacy Dashboard
  2. Tap "Delete All My Data"
  3. Confirm with biometric authentication (fingerprint/face) or passcode
  4. All local data is permanently erased from Android storage (cannot be recovered)

Note: If you have Google Drive backups enabled, you must separately delete backups from Google Drive by going to drive.google.com ‚Üí Backups ‚Üí EZer ‚Üí Delete.

14. Security Measures (Android)

14.1 Encryption (Android Keystore)

  • Database: SQLCipher with AES-256-GCM encryption
  • Keys: Stored in Android Keystore (hardware-backed on Pixel, Samsung flagship devices)
  • Receipt Images: Encrypted before storage in app cache
  • Backups: End-to-end encrypted before Google Drive upload
  • SharedPreferences: EncryptedSharedPreferences with Android Keystore

14.2 Authentication (Android Biometrics)

  • Biometric authentication using Android BiometricPrompt API (fingerprint, face unlock, iris)
  • App lock with automatic timeout
  • Session management with auto-logout
  • Support for Android StrongBox Keymaster on compatible devices

14.3 Root Detection (freeRASP for Android)

üîç Android Security Framework

Using freeRASP security framework, EZer detects:

  • Rooted Android devices (Magisk, SuperSU, KingRoot, Magisk Delta)
  • Custom ROMs with su binaries
  • App tampering and repackaging (APK signature verification)
  • Debugging attempts (USB debugging, JDWP)
  • Emulator/virtual machine environments (BlueStacks, Genymotion)
  • Xposed Framework or Frida hooks

Security warnings are displayed if risks are detected. Some features may be restricted on rooted devices.

14.4 Certificate Pinning (Android Network Security)

If you enable optional cloud features, EZer uses certificate pinning via Android Network Security Config to prevent man-in-the-middle attacks on network connections.

14.5 Secure Development Practices

  • Regular security audits and code reviews
  • Android Gradle dependency vulnerability scanning
  • Secure coding standards (OWASP Mobile Top 10)
  • Penetration testing before major releases
  • ProGuard/R8 code obfuscation enabled in release builds
