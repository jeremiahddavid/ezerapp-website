1. Introduction

EZer ("we", "our", "us", or "the App") is a personal finance management application developed for iOS devices. EZer helps you track expenses, manage budgets, scan receipts, import transactions from files, and gain insights into your financial health.

This Privacy Policy explains:

  • What personal data we collect and why
  • How we store and protect your data on iOS
  • iOS-specific permissions (camera, photo library, notifications)
  • Your rights regarding your personal information
  • How to exercise your privacy rights

By downloading, installing, or using EZer on iOS, you agree to this Privacy Policy. If you do not agree, please do not use the App.

2. Information We Collect

EZer collects different types of information depending on the features you choose to use. All data collection requires your explicit consent.

2.1 Financial Data (Required for Core Functionality)

When you use EZer, we collect and store the following financial information locally on your iOS device:

  • Transactions: Date, amount, merchant name, category, payment method, notes
  • Accounts: Account names, types (bank, wallet, credit card), balances
  • Budgets: Budget categories, limits, time periods, spending alerts
  • Bills: Biller names, amounts, due dates, payment history, recurrence settings
  • Goals: Savings goal names, target amounts, deadlines, progress tracking
  • Categories: Custom categories and subcategories you create
  • Receipts: Images of receipts you scan or attach to transactions

2.2 Transaction Import Methods (iOS)

üì± iOS Platform Note

Unlike Android, iOS does NOT allow apps to read SMS messages. This is an Apple platform restriction to protect user privacy. All transactions must be entered manually or imported from files.

Transaction entry methods available on iOS:

  • Manual Entry: Quick transaction entry with smart categorization and merchant detection
  • CSV Import: Import bank statement CSV files with automatic field mapping
  • PDF Import: Extract transactions from bank PDF statements using on-device OCR
  • Excel Import: Import transactions from Excel files (.xlsx, .xls)
  • Receipt Scanning: Capture receipts with camera and extract transaction details

‚úÖ Privacy Benefit

The iOS version of EZer is even more privacy-focused than Android since there's no SMS access at all. All transaction data is entered manually or imported from files you explicitly choose.

2.3 Camera and Storage Access (Optional)

If you enable receipt scanning features, we request:

  • Camera Access: To capture photos of receipts
  • Photo Library/Storage Access: To select existing receipt images from your device (iOS 13+ uses limited photo access)

Receipt images are stored encrypted in your device's local database and processed using on-device OCR (Optical Character Recognition) powered by Apple Vision Framework. No images are uploaded to external servers.

2.4 Notifications (iOS Only)

üîî iOS Notifications

EZer can send you notifications for bill reminders, budget alerts, and financial insights. Unlike Android, iOS does NOT allow apps to read other apps' notifications. All EZer notifications are outbound only (sent by EZer to you).

2.5 Device Information (Automatically Collected)

We collect minimal technical information for app functionality and troubleshooting:

  • Device model and manufacturer (e.g., iPhone 15 Pro, iPad Pro)
  • iOS OS version (e.g., iOS 17)
  • App version number
  • Device language and region settings
  • Screen resolution and display metrics (for UI optimization)

This information does not include device identifiers like IMEI, serial numbers, or iOS Advertising ID.

2.6 Usage Analytics (Optional - Opt-In Only)

If you explicitly enable analytics in Settings, we collect anonymous usage data:

  • Screen views and navigation patterns (e.g., "Budget screen viewed")
  • Feature usage frequency (e.g., "Receipt scan used 5 times")
  • App performance metrics (loading times, errors)
  • Crash reports with stack traces

üö´ What we DO NOT collect in analytics:

  • Transaction amounts or balances
  • Merchant names or transaction descriptions
  • Account names or numbers
  • Any personally identifiable financial data

2.7 Biometric Data (Processed Locally)

If you enable biometric authentication (fingerprint, face unlock), biometric data is:

  • Processed entirely by iOS's LocalAuthentication API and iOS Keystore
  • Never accessed, stored, or transmitted by EZer
  • Used only to unlock the app - we never receive the actual biometric data

3. How We Store Your Data (iOS)

üõ°Ô∏è

Local-First, Encrypted Storage on iOS

Your financial data never leaves your iOS device unless you explicitly choose to export it or enable cloud backup. All data is stored in an encrypted SQLite database using SQLCipher encryption with iOS Keystore.

3.1 Database Encryption (iOS Keystore)

EZer uses industry-standard encryption on iOS:

  • Database Encryption: SQLCipher with AES-256-GCM encryption
  • Encryption Keys: Stored in iOS Keystore (hardware-backed on supported devices)
  • Key Generation: Cryptographically secure random key generation using iOS SecureRandom
  • No Master Password: Encryption keys are device-specific and bound to iOS Keystore
  • StrongBox Support: Uses hardware security module on compatible devices (iPhone, iPad Pro)

3.2 Secure Storage Locations (iOS)

  • Database: App-specific private storage (/data/data/com.ezer.personalfinance/databases/) - inaccessible to other apps
  • Encryption Keys: iOS Keystore (system-level secure enclave)
  • Receipt Images: Encrypted and stored in app's private cache directory
  • Shared Preferences: Encrypted using Keychain Services with iOS Keystore

3.3 Jailbreak Detection (iOS)

üîç Security Checks

EZer implements security checks using freeRASP to detect:

  • Jailbroken iOS devices (Checkra1n, unc0ver, Taurine)
  • Custom ROMs with su binaries
  • App debugging and tampering
  • Emulator/virtual machine environments
  • Xposed Framework or Frida hooks

If security risks are detected, EZer will warn you and may restrict sensitive features to protect your data.

‚ö†Ô∏è

We Do NOT Sell Your Data

EZer does not sell, rent, or share your personal financial data with third parties. Period. Your data remains on your iOS device and under your control at all times.

5.1 When Data May Leave Your iOS Device

If you enable crash reporting:

We may disclose information if required by law, such as in response to valid court orders or subpoenas.

6. iOS Platform Limitations

‚ö†Ô∏è

Important iOS Platform Information

Unlike the Android version, EZer for iOS does NOT have automatic SMS parsing or notification reading capabilities.

This is an iOS platform restriction designed by Apple to protect user privacy. All transactions must be entered manually or imported from bank statements (CSV, PDF, Excel). This makes iOS even more privacy-focused than Android.

6.1 Why SMS Reading is Not Available on iOS

Apple does not allow third-party apps to access SMS messages for security and privacy reasons. This is a deliberate design choice by Apple to protect user privacy.

6.2 Alternative Transaction Entry Methods on iOS

Instead of automatic SMS import, EZer for iOS offers:

  • Manual Entry: Quick and easy transaction entry with smart categorization
  • CSV Import: Import bank statement CSV files
  • PDF Import: Extract transactions from bank PDF statements using on-device OCR
  • Excel Import: Import transactions from Excel files
  • Receipt Scanning: Capture receipts with camera and extract transaction details

6.3 Privacy Benefits of iOS Restrictions

‚úÖ Enhanced Privacy on iOS

The iOS platform restriction actually enhances your privacy by ensuring that no app (including EZer) can ever access your SMS messages. Your banking SMS remain completely private to you alone.

7. Notification Permissions (iOS)

7.1 In-App Notifications

EZer sends local notifications on iOS for:

  • Bill Reminders: Upcoming bill due dates (using iOS UNUserNotificationCenter)
  • Budget Alerts: When you approach or exceed budget limits
  • Goal Milestones: Progress updates on savings goals
  • Spending Insights: Weekly or monthly spending summaries

All notifications are generated locally using iOS's UNNotificationCenter. You can customize notification channels in iOS Settings ‚Üí Apps ‚Üí EZer ‚Üí Notifications.

7.2 Notification Listener (Optional - iOS Only)

üîî Advanced iOS Feature

If you enable automatic transaction detection from banking app notifications:

  • EZer can read notifications from banking and payment apps
  • Transaction notifications are parsed locally to auto-create transactions
  • Notification content is never transmitted to external servers
  • Only financial app notifications are processed - other app notifications are ignored
  • Requires BIND_NOTIFICATION_LISTENER_SERVICE permission (granted via iOS Settings)

8. Camera and Storage Permissions (iOS)

8.1 Camera Access (iOS)

Camera permission (android.permission.CAMERA) is used exclusively for:

  • Capturing receipt photos using iOS AVFoundation
  • Scanning documents related to financial transactions

EZer does not access your camera for any other purpose, nor does it record videos or access photo metadata (EXIF data).

8.2 Photo Library Access (iOS Storage)

Storage permission allows you to:

  • iOS 13+: Uses limited photo access (PHPhotoLibrary) - access only to selected images
  • iOS 12 and below: READ_EXTERNAL_STORAGE for selecting receipt images
  • Export financial reports to Downloads folder
  • Save backup files locally

8.3 Receipt Image Processing (On-Device OCR)

When you scan a receipt on iOS:

  1. Image is captured using iOS Camera API or selected from photo library
  2. OCR processing happens on-device using Apple Vision Framework (no cloud upload)
  3. Extracted text (merchant, amount, date) is saved to your encrypted SQLite database
  4. Receipt image is encrypted and stored in app's private storage
  5. Original photo in your photo library is not modified or deleted

9. Machine Learning and AI Features (On-Device)

9.1 On-Device Machine Learning (Core ML)

EZer uses machine learning on your iOS device for:

  • Automatic Categorization: Suggesting categories for transactions based on merchant names
  • Spending Insights: Identifying unusual spending patterns and trends
  • Budget Predictions: Forecasting future spending based on historical data
  • OCR Enhancement: Improving receipt text recognition accuracy

9.2 Privacy-Preserving ML (Core ML on iOS)

ü§ñ On-Device AI

All machine learning in EZer is privacy-preserving and runs entirely on your iOS device:

  • ML models run using Core ML (optimized for iOS)
  • Your transaction data is never sent to external servers for training
  • Pre-trained models are included in the IPA
  • Personalized learning happens locally using on-device data
  • Uses iOS NNAPI for hardware acceleration on supported devices

10. Cloud Backup (Optional - iCloud)

10.1 How iCloud Backup Works on iOS

If you enable cloud backup:

  1. You authenticate with iCloud using Google Sign-In
  2. EZer creates an encrypted backup of your SQLite database
  3. Backup is encrypted using your iOS Keystore encryption key before upload
  4. Encrypted backup is stored in your personal iCloud account
  5. Only you can decrypt and restore the backup on your iOS device

10.2 What's Included in iOS Backups

  • All transaction records from SQLite database
  • Accounts, budgets, bills, goals
  • Categories and custom settings
  • Receipt images (encrypted)
  • App preferences (SharedPreferences)

10.3 Backup Security (iOS Keystore)

  • Backups are encrypted using AES-256 before upload
  • Encryption key remains in iOS Keystore (never uploaded)
  • Google cannot decrypt your backups
  • Backups are accessible only through your authenticated Google account

  • Export your data in multiple formats

12. Data Retention and Deletion (iOS)

12.1 How Long We Retain Data on iOS

  • Financial Data: Retained on your iOS device until you delete it or uninstall the app
  • Receipt Images: Stored in app cache until you delete the associated transaction or clear cache
  • Consent Logs: Audit logs retained for 2 years for compliance purposes
  • iCloud Backups: Retained in your iCloud until you delete them

12.2 Automatic Data Cleanup on iOS

EZer automatically cleans up:

  • Temporary OCR cache files (cleared after 7 days)
  • Deleted transactions (permanently removed after 30 days)
  • Old notification logs (kept for 90 days for audit purposes)

12.3 Permanent Data Deletion on iOS

To permanently delete all your data from your iOS device:

  1. Open EZer ‚Üí Settings ‚Üí Privacy Dashboard
  2. Tap "Delete All My Data"
  3. Confirm with biometric authentication (fingerprint/face) or passcode
  4. All local data is permanently erased from iOS storage (cannot be recovered)

Note: If you have iCloud backups enabled, you must separately delete backups from iCloud by going to drive.google.com ‚Üí Backups ‚Üí EZer ‚Üí Delete.

If you are a parent or guardian and believe your child has provided us with personal information, please contact us at privacy@ezerapp.com. We will promptly delete such information.

14. Security Measures (iOS)

14.1 Encryption (iOS Keystore)

  • Database: SQLCipher with AES-256-GCM encryption
  • Keys: Stored in iOS Keystore (hardware-backed on iPhone, iPad Pro devices)
  • Receipt Images: Encrypted before storage in app cache
  • Backups: End-to-end encrypted before iCloud upload
  • SharedPreferences: Keychain Services with iOS Keystore

14.2 Authentication (iOS Biometrics)

  • Biometric authentication using iOS LocalAuthentication API (Face ID, Touch ID)
  • App lock with automatic timeout
  • Session management with auto-logout
  • Support for iOS StrongBox Keymaster on compatible devices

14.3 Jailbreak Detection (freeRASP for iOS)

üîç iOS Security Framework

Using freeRASP security framework, EZer detects:

  • Jailbroken iOS devices (Checkra1n, unc0ver, Taurine, Odyssey)
  • Custom ROMs with su binaries
  • App tampering and repackaging (IPA signature verification)
  • Debugging attempts (USB debugging, JDWP)
  • Emulator/virtual machine environments (BlueStacks, Genymotion)
  • Xposed Framework or Frida hooks

Security warnings are displayed if risks are detected. Some features may be restricted on jailbroken devices.

14.4 Certificate Pinning (iOS Network Security)

If you enable optional cloud features, EZer uses certificate pinning via iOS Network Security Config to prevent man-in-the-middle attacks on network connections.

14.5 Secure Development Practices

  • Regular security audits and code reviews
  • iOS Gradle dependency vulnerability scanning
  • Secure coding standards (OWASP Mobile Top 10)
  • Penetration testing before major releases
  • Swift code obfuscation enabled in release builds

15. App Tracking Transparency (iOS 14.5+)

‚úÖ

No Tracking - ATT Not Required

EZer does NOT track you across apps or websites. We do not use IDFA (Identifier for Advertisers) or any cross-app tracking. Therefore, we do not display Apple's App Tracking Transparency (ATT) prompt. Your activity in EZer is completely private.

15.1 What is App Tracking Transparency?

Starting with iOS 14.5, Apple requires apps to ask for user permission before tracking user activity across apps and websites. Apps must use the AppTrackingTransparency framework to request permission.

15.2 EZer Does Not Track Users

EZer respects your privacy and does not engage in any tracking activities:

  • No IDFA Collection: We do not access your device's advertising identifier
  • No Cross-App Tracking: We do not track your activity across other apps
  • No Ad Networks: EZer does not integrate with advertising networks
  • No Data Brokers: We do not share your data with data brokers
  • No Third-Party Analytics: We do not send your data to third-party analytics platforms

15.3 App Store Privacy Nutrition Labels

In accordance with Apple's Privacy Nutrition Labels, EZer declares:

  • Data Not Linked to You: All data collected is stored locally and not linked to your identity
  • Data Not Used to Track You: We do not use your data for tracking purposes
  • No Data Shared with Third Parties: Your financial data never leaves your device
